# REVIEW

## VERDICT
FAIL
Phase 3 stays Pi-native and phase-scoped, but the current audit-state restoration and abandoned-branch workflow are not correct enough to approve. `npm run build`, `npm test`, and `vitest run tests/pi-extension.integration.test.ts` all passed on rerun; the blocking issues are logic and coverage defects, not build breakage.

## BUGS
- `.pi/extensions/ozcar/state/audit-session.ts:154-163` unconditionally overlays the latest labeled `branch_summary` onto the latest `ozcar-audit-state`, even when a newer custom state exists later on the same branch. I verified this with a runtime repro using a branch `root state -> labeled branch_summary -> new root state`: `restoreAuditStateFromBranch()` returned `auditId: "new-audit"` with `branchKind: "abandoned"` plus the stale summary, so `session_start` and `session_tree` can restore the wrong branch after starting or resuming work from a summarized branch.
- `.pi/extensions/ozcar/commands/audit.ts:95-109`, `.pi/extensions/ozcar/tools/audit-branch.ts:26-40`, and `.pi/extensions/ozcar/state/audit-session.ts:256-262` disagree about abandoned branches. The command and tool surfaces accept `abandoned` as a normal checkpoint, but the `session_before_tree` hook explicitly stops generating abandoned-summary labels once the current state is already `abandoned`. I verified this with a runtime repro: `buildAuditTreeConventionResult()` returned `undefined` for an `abandoned` current state, so the summarized `/tree` exit loses the `audit:abandoned:<slug>` label and the restoreable abandoned summary state the Phase 3 contract relies on.

## GAPS
- `tests/pi-extension.integration.test.ts:263-279` and `tests/pi-extension.integration.test.ts:481-499` only prove reload and `navigateTree(..., { summarize: false })` back to the root audit state. There is no end-to-end proof for the summarized `/tree` path that should create a labeled `branch_summary` and then restore that state on `session_tree` or `session_start`, which is exactly where the restoration bug lives.
- `tests/pi-extension.audit.test.ts:293-350` covers stale labels and abandoned-summary restoration independently, but it does not cover the combined sequence `labeled branch_summary -> later new audit state` or the user-facing `abandoned` command/tool flow before a summarized `/tree` exit.

## ANTI-PATTERNS
None

## IMPROVEMENTS
- Add regression coverage for `restoreAuditStateFromBranch()` precedence when a newer `ozcar-audit-state` exists after a labeled `branch_summary`.
- Add an integration proof for summarized `/tree` navigation that exercises `session_before_tree`, branch-summary label placement, `session_tree`, and reload restoration in one flow.
- Make the abandoned-branch contract explicit in the command and tool surfaces: either remove direct `abandoned` checkpoints from the user-facing API or make them feed the summary-labeling path instead of disabling it.

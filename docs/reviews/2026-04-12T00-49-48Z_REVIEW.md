# REVIEW

## VERDICT
PARTIAL
Phase 2 stays within the current `PLAN.md` scope and the recorded execution artifact mostly matches the checked-in code, but the plan-to-scan provider snapshot is not durable across a resumed or separately invoked phase boundary.

- Scope check: the implementation remains TypeScript-only, keeps JSON contracts canonical, adds the planned provider adapters, prompts, phases, gates, and the minimal `run --dry-run` seam, and does not pull Phase 3+ behavior forward beyond reserved command placeholders.
- Validation rerun: `npm run build` passed; `npm test` passed (`3` files, `14` tests); the recorded dry Phase 2 command passed and emitted `plan.json`, `plan.prompt.md`, `scans/0001/request.json`, `scans/0001/prompt.md`, and `scans/0001/output.json`; `npx vitest run tests/phase2.test.ts -t 'fails the scan gate deterministically when output.json is missing'` passed.
- Extra verification: the CLI auto-detection path also passed in a fresh dry run without `--provider`, selected `codex` on the current `PATH`, and emitted the expected loop `0001` artifacts.
- Reproduced drift: calling the built `runPlanPhase()` with `model: "custom-phase2-model"` and then `runScanPhase()` without a model rewrote the durable provider snapshot from `plan.provider.model = "custom-phase2-model"` to `request.provider.model = "claude-default"`. Repeating the same seam with a `claude` plan provider and a `codex` scan provider produced `plan.provider.name = "claude"` but `request.provider.name = "codex"`.

## BUGS
- High: the scan phase rebuilds provider identity from fresh inputs instead of treating `plan.json` as the canonical cross-phase contract. `runPlanPhase()` persists the chosen provider/model in `src/phases/plan.ts:46-68`, but `runScanPhase()` recalculates and writes a new provider snapshot from `options.provider` and `options.model` in `src/phases/scan.ts:42-72`. That allows resumed or separately invoked scans to emit replay artifacts that disagree with the loop plan on provider name, selection mode, or model, which breaks the Phase 2 resumability and replayability contract.

## GAPS
- There is no deterministic phase-boundary check or seam test that asserts `scans/*/request.json` preserves the provider snapshot from `plan.json`. `assertScanGate()` only checks run/loop/scan identity, and the current Phase 2 tests cover provider resolution and single-path artifact emission but not cross-phase provider/model continuity.

## ANTI-PATTERNS
None

## IMPROVEMENTS
- Make `plan.json` the canonical source for scan-provider metadata. Either derive the scan request prompt/rendered provider fields directly from `options.plan.provider`, or reject mismatches before writing artifacts, then add a focused seam test that invokes `runPlanPhase()` and `runScanPhase()` separately with a different provider or model and expects deterministic failure.

Re-ran `npm test -- tests/audit-artifacts.test.ts tests/pi-extension.export.test.ts tests/pi-extension.test.ts`, `npm run build`, and `npm test`; all passed. ## BUGS
- `src/reports/export-findings.ts:44-68`: `deriveFindingsExportTimestamp()` folds timestamps from every bundle in the snapshot, including pending and rejected findings. That lets non-validated churn change `exports/findings.json` bytes even when the exported validated findings are unchanged. This contradicts the validated-only comparison contract surfaced by `.pi/extensions/ozcar/commands/export.ts:52-55` and `README.md:60-65`, and it weakens the “pinned stable export bytes” claim in Phase 6. I verified it by rebuilding a synthetic snapshot through `dist/reports/export-findings.js`: changing only a pending bundle’s `validatedAt` moved `generatedAt` from `2026-04-12T20:08:00.000Z` to `2026-04-12T20:09:00.000Z` while the serialized `findings` array remained identical.

## GAPS
- `tests/audit-artifacts.test.ts:156-196` pins canonical timestamp normalization and fixture bytes, but it never mutates only a non-validated bundle timestamp and asserts unchanged export bytes. That missing case let the Phase 6 stability regression above pass the recorded proof.

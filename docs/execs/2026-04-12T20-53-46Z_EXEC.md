# EXECUTION

## PLAN

- Implement the first Phase 3 Pi-native audit slice inside `.pi/extensions/ozcar` without extending the transition-era CLI runtime under `src/`.
- Add slash commands for audit start or resume plus current-state inspection, backed by session entries that reconstruct on `session_start` and `session_tree`.
- Define tree-aware audit conventions for hypotheses, confirmed findings, and abandoned branches through Pi labels and branch summaries instead of a parallel store.
- Update the repo-local prompt, skill, and README surface so the new Phase 3 workflow is discoverable from Pi.
- Validate the slice with unit coverage, Pi integration coverage, `npm run build`, `npm test`, and a Phase 3 Pi proof when the local `pi` binary is available.

## CHANGES

- Expanded the project-local Pi extension under `.pi/extensions/ozcar/` from a single scaffold file into focused Phase 3 modules:
  - `index.ts` now wires the Phase 3 command surface, the `ozcar_audit_branch` tool, `session_start` and `session_tree` restoration, and `session_before_tree` abandoned-branch conventions.
  - `types.ts` defines the minimal Pi extension contracts needed for local type-checking and tests without pulling a repo-owned Pi runtime into `src/`.
  - `commands/audit.ts` adds `/ozcar-audit-start`, `/ozcar-audit-resume`, `/ozcar-audit-state`, and `/ozcar-audit-branch`.
  - `state/audit-session.ts` adds the session-backed audit-state helper that persists branch-local `ozcar-audit-state` custom entries, reconstructs state from the active branch, updates Pi session UI metadata, and derives abandoned-branch summary conventions for `/tree`.
  - `state/labels.ts` defines the Phase 3 label and summary conventions for `audit:hypothesis:<slug>`, `audit:confirmed:<finding-id>`, and `audit:abandoned:<slug>`.
  - `tools/audit-branch.ts` adds the minimal LLM-callable tool for checkpointing hypotheses, confirmed findings, and abandoned paths against the current Pi branch.
- Updated the repo-local Pi entry surfaces so the new workflow is discoverable:
  - `.pi/prompts/ozcar-audit.md` now calls out `/ozcar-audit-start`, `/ozcar-audit-state`, `/ozcar-audit-resume`, and branch checkpointing.
  - `.pi/skills/ozcar-audit/SKILL.md` now documents the first coherent Phase 3 slice instead of the old Phase 2 scaffold only.
  - `README.md` now describes the Phase 3 Pi-first quickstart, the new slash commands, and the tree-label conventions.
- Added and updated tests for the touched Phase 3 seams:
  - `tests/pi-extension.test.ts` now checks the richer slash-command surface and the registered Phase 3 audit tool.
  - `tests/pi-extension.audit.test.ts` adds unit coverage for start and branch argument parsing, command behavior, session-backed state restoration, abandoned-summary convention generation, stale-label handling, tool behavior, and restoration on `session_start` plus `session_tree`.
  - `tests/pi-extension.integration.test.ts` now covers the isolated Phase 3 command surface in a real `pi` RPC subprocess, reload restoration, and tree-navigation restoration with helper-driven branch dumps.

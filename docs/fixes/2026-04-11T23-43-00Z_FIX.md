# FIX

## PLAN

- Root cause 1: the run-store validates `run.json` and `events.jsonl` as independent schema-shaped files, but it never verifies that they still belong to the requested `<run-id>` directory.
- Root cause 2: the Phase 1 seam tests only covered happy-path create/reopen/lock/event flows, so corruption of store identity was never exercised.
- Repair the shared read path in `src/store/run-store.ts` and `src/store/events.ts` so `status` and `resume` reject mismatched `run.json` and mismatched event histories with one store-level invariant check.
- Add regression tests that corrupt `run.json` and `events.jsonl` and assert that snapshot and reopen paths fail closed.

## CHANGES

- Added explicit run identity enforcement in [src/store/run-store.ts](/Users/x/oz/ozcar/src/store/run-store.ts) so `readRunRecord()`, `readRunSnapshot()`, and `reopenRunStore()` reject a `run.json` whose `runId` no longer matches the requested run directory.
- Added explicit event identity enforcement in [src/store/events.ts](/Users/x/oz/ozcar/src/store/events.ts) so `readEventRecords()` rejects any `events.jsonl` line whose `runId` does not match the expected store.
- Added corruption regression coverage in [tests/run-store.test.ts](/Users/x/oz/ozcar/tests/run-store.test.ts) for both mismatched `run.json` and mixed-run `events.jsonl`, and asserted that both snapshot and reopen paths fail closed.

## PLAN

- Root cause 1: the lock seam treats any pre-existing `.lock/` directory as active contention, so a dead-process lock can block `resume` indefinitely even when the recorded owner is no longer alive on the same host.
- Root cause 2: the prior fix artifact at [docs/fixes/2026-04-11T23-43-00Z_FIX.md](/Users/x/oz/ozcar/docs/fixes/2026-04-11T23-43-00Z_FIX.md) overstated `run.json` enforcement by implying already-open handles reread `run.json`; the code only revalidates `run.json` on snapshot and reopen, while open handles validate `events.jsonl` reads.
- Repair the lock acquisition path in `src/store/lock.ts` so same-host stale locks with dead owner pids are reclaimed before failing with `RunLockError`.
- Add focused regression coverage for stale lock reclamation and correct the inaccurate sentence in the older fix artifact without rewriting its rest of record.

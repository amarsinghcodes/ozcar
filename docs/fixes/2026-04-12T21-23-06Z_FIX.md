# FIX

## PLAN

- Root cause 1: `restoreAuditStateFromBranch()` treated a late labeled `branch_summary` as globally authoritative, even when a newer `ozcar-audit-state` existed later on the same branch, so stale abandoned summaries could override newer root state.
- Root cause 2: the user-facing branch checkpoint surface allowed direct `abandoned` state while the `/tree` summary hook only handled non-abandoned active state, so summarized exits and explicit branch checkpointing diverged.
- Root cause 3: the validation proved reload and non-summarized tree navigation only, leaving the summarized `/tree` path and post-summary restoration untested end to end.
- Repair group 1: make restoration follow branch order, persist abandoned summary state on summarized `session_tree` exits, and keep only a bounded backward-compatible fallback for older summary-only branches.
- Repair group 2: narrow the command and tool checkpoint surface to active hypothesis and confirmed branches, leaving abandoned state to the summary-driven `/tree` path.
- Repair group 3: add regression tests for stale summary precedence, the summarized `/tree` flow, and post-summary restoration on reload and tree navigation.

## CHANGES

- `.pi/extensions/ozcar/state/audit-session.ts` now restores audit state by branch order instead of letting any latest labeled `branch_summary` win globally. The latest `ozcar-audit-state` custom entry is authoritative, matching labels are attached only to that state, and a labeled `branch_summary` only overlays state when it is later than the newest custom state on the same branch.
- `.pi/extensions/ozcar/index.ts` now persists a fresh `ozcar-audit-state` checkpoint on `session_tree` when Pi emits a labeled abandoned `summaryEntry`, then re-syncs runtime state through the shared restoration path. This makes summarized `/tree` exits restoreable on `session_start`, `session_tree`, and reload without relying on summary-only fallback forever.
- `.pi/extensions/ozcar/commands/audit.ts`, `.pi/extensions/ozcar/tools/audit-branch.ts`, `README.md`, `.pi/prompts/ozcar-audit.md`, and `.pi/skills/ozcar-audit/SKILL.md` now treat `/ozcar-audit-branch` and `ozcar_audit_branch` as active checkpoint surfaces for `hypothesis` and `confirmed` only. Abandoned state is now explicitly summary-driven through Pi's tree flow instead of a separate direct checkpoint mode.
- `tests/pi-extension.audit.test.ts`, `tests/pi-extension.integration.test.ts`, and `tests/pi-extension.test.ts` now cover direct rejection of `abandoned` on the command/tool surface, stale `branch_summary -> later new audit state` precedence, `session_tree` summary-state persistence, reload restoration, and a live Pi summarized-tree proof that also verifies a later root state beats an older abandoned summary on the same branch.

# FIX

## PLAN

- Root cause 1: the live RPC proof in `tests/pi-extension.integration.test.ts` still bypassed Pi's default branch summarizer and ozcar's `session_before_tree` label path by injecting both the summary text and abandoned label from the helper extension.
- Root cause 2: RPC mode cannot drive Pi's built-in `/tree` selector, so the plain interactive `/tree` path stayed unproven end to end even after the summarized branch restoration logic was fixed.
- Repair group 1: remove the helper-supplied abandoned label and summary from the RPC integration proof so the automated live proof exercises ozcar's real `session_before_tree` conventions plus Pi's default summarizer.
- Repair group 2: add a separate live interactive proof harness for the TUI `/tree` selector that can run when `pi`, `expect`, and a supported model credential are available locally.
- Repair group 3: keep the Phase 3 validation artifact explicit about which proof runs are automated, which are environment-gated live proofs, and any remaining model-path risk.

## CHANGES

- `tests/pi-extension.integration.test.ts` now runs the live summarized-tree proof through ozcar's real `session_before_tree` label path and Pi's default branch summarizer. The helper extension still provides dump and root-navigation seams, but it no longer injects the abandoned label or the summary text.
- `tests/pi-extension.interactive.test.ts` now adds a gated live TUI proof for the plain interactive `/tree` selector. It drives Pi through `expect`, searches the real tree for the two `ozcar-audit-state` checkpoints, selects the root checkpoint through the selector, accepts Pi's built-in summary prompt, and verifies the abandoned branch state from the resulting `session_tree` branch dump.
- `README.md` now lists the Phase 3 live-proof commands so the stronger RPC proof and the gated interactive `/tree` proof are discoverable from the repo root.

## RISKS

- The interactive `/tree` proof is intentionally environment-gated because it depends on local `pi`, `expect`, and a usable live model credential. The default suite therefore keeps that proof skipped unless `OZCAR_RUN_INTERACTIVE_TREE_PROOF=1` is set.
- The expect-driven TUI proof matches current Pi screen text and key flow, so upstream changes to the `/tree` selector copy or keybindings could require a proof-harness update even when the underlying Phase 3 behavior is still correct.

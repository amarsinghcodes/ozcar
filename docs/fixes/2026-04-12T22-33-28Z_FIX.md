# FIX

## PLAN

- Root cause 1: the Phase 4 store reader validates JSON shape but never binds the embedded `auditId` and `findingId` values back to the requested audit root or containing finding directory, which lets corrupted on-disk ids redirect rebuild output or hide renamed finding directories.
- Root cause 2: the timestamp contract accepts any parseable timestamp string without canonicalizing it, so equivalent instants can survive with different byte encodings and produce nondeterministic export JSON.
- Repair group 1: enforce requested-audit-root and finding-directory identity during store reads, keep rebuild and freshness paths pinned to the requested audit id, and fail closed when stored contracts drift from their filesystem location.
- Repair group 2: canonicalize timestamps at the shared schema boundary and rebuild export timestamps from normalized values only.
- Repair group 3: add adversarial tests for audit-root drift, finding-directory drift, canonical timestamp normalization, and missing-artifact freshness.

## CHANGES

- Enforced audit-root and bundle-path identity in `src/artifacts/store.ts`:
  - added `AuditArtifactStoreError` so missing, invalid, and identity-drift failures stay typed and carry the offending path
  - `readAuditArtifactSnapshot()` now pins reads to the requested `auditId`, rejects `audit.json` and `scope.json` whose embedded `auditId` does not match the requested audit root, and rejects finding bundles whose containing directory name does not match the embedded `findingId`
  - `rebuildAuditArtifacts()` and `inspectAuditArtifactFreshness()` now keep output paths pinned to the requested audit root instead of re-deriving them from stored embedded ids
- Canonicalized timestamps at the shared contract seam:
  - `src/contracts/run.ts` now normalizes accepted ISO-8601 timestamps to `Date#toISOString()` form during schema parse instead of preserving arbitrary parseable strings
  - `src/reports/export-findings.ts` now parses the incoming snapshot before deriving `generatedAt`, so export timestamps are computed from normalized values only
- Expanded the Phase 4 adversarial coverage in `tests/audit-artifacts.test.ts`:
  - added a regression test for embedded audit-id drift causing rebuild redirection
  - added a regression test for finding-directory identity drift during freshness inspection
  - added a regression test for equivalent offset-vs-UTC timestamp encodings producing identical stored and exported bytes
  - added a regression test for missing `summary.md`, `confirmed-findings.md`, and `exports/findings.json` freshness reporting
